# 多开残留双跑窗口:根因是读取面粒度用错,不是「彻底闭合太贵」

## 现象

`multiOpenSeen`(前一条修法)留下的「残留窗口」不是低概率时序,而是**确定性**的:先起跑的
一侧只要还有第二波用例、且对方比它快,第二波里被对方跑掉的**每一条**都会被重跑。

冒烟复现(全量重叠 + 双方各有第二波,fake agent + fake sandbox、同一个 `niceevalRoot`、
4 条用例各 2 并发):A 认领 `e1,e2` 挂住,B 撞锁挂起后认领 `e3,e4` 跑完并释放锁,再放行 A ——
A 的第二波 `e3,e4` 干净取到空锁,全局真实派发序列长度 6(`e3=2 e4=2`)。

## 根因

`multiOpenSeen` 的六条判据全是「**此刻**碰上了对方」或「对方**死**了」的证据(启动时锁目录
非空 / 取锁前读到别人的记录 / 撞过锁 / 接管过 / 等过实验闸名额 / 接管过租约),没有任何一条
能观测到「对方**干净地来过又走了**」。而「对方来过」的唯一证据是结果树上新落盘的
`result.json`,那恰恰只有重查会去读——鸡生蛋。先起跑的一侧天然全程零信号,于是裸奔。

它之所以只能是启发式,是因为重查用 `loadLatestResultsPerEval` **全根扫描**去回答一个
per-case 的问题(「这条用例现在还缺哪些 attempt」)。实测 110 ms/条,而且这次 I/O 是在握着
全局并发位 + 实验闸名额 + 用例锁的状态下付的,60 条用例 ≈ 6.6 s 纯串行 I/O 捂着别人在等的
名额——贵到只能靠启发式去省,省着省着就漏了。

顺带的真实副作用:判据 1 把「上一次被强杀留下的遗留锁条目」也算成多开,于是单开重跑一个上次
被强杀的批次时 `multiOpenSeen` 恒真,**每条用例都做一次全树扫描**。加护栏反而更慢又更不安全。

## 修法(已修,commit d3963179)

把读取面收窄,然后**无条件**重查——闭合性来自 `对方 result.json 落盘 → 对方释放锁 →
我们取到锁 → 我们读盘` 这条 happens-before 链,所以重查必须在取锁**之后**,顺序反了什么都
不保证。

- `src/results/open.ts` 新增 `loadLatestResultsForCase(root, experimentId, evalId)`:只 readdir
  该实验目录、逐快照试该 eval 的 attempt 目录,没跑过这条 eval 的快照不读任何文件。
- `src/runner/fingerprint.ts` 抽出 `carriableAttempts`,`planCarry` 与重查共用同一份判据。
- `src/runner/run.ts` 删掉 `multiOpenSeen` 全部六个置真点,取到锁一律 `recheckCarry`。

实测(MemoryBench,4 实验 / 38 快照 / 720 份 `result.json` / 119 MB):全树 232 ms 冷、
~110 ms 热;收窄读单条 p50 **0.89 ms** / p90 1.27 ms,未命中(该实验从没跑过这条 eval)
0.23–0.38 ms。逐条比对 144 个 `(experiment, eval)` 组合,收窄读与全树扫描的
verdict / `artifactBase` / `locator` 完全一致。

## 两条反直觉点

**1. 收窄读不是「更快的全量读」,别顺手拿去替换启动时那次整批读。** 直觉上「收窄读快 100 倍,
那启动时也用它」——实测相反:同一棵树上逐条收窄读全部 144 个组合合计 **141 ms**,一次全树
扫描只要 110 ms(热)。收窄读是 O(每条用例 × 该实验快照数) 次 readdir,全量场景下这个乘积
会翻回来。它的定位是「派发路径上的 per-case 问题」,`src/cli.ts` 启动时那次
`loadLatestResultsPerEval` 是 per-invocation 的整批问题,保持全树扫描才对。

**2. 写双开冒烟时,「对方 started」不等于「对方释放了锁」。** 锁释放发生在该用例全部 attempt
收尾之后。barrier 放早了,己方第二波撞上的是**新鲜锁** → 走挂起窗口,而挂起窗口这条路径本来
就重查携带——于是测试绿得毫无意义,根本没碰到 clean-acquire 这个残留窗口。必须先等锁目录
条目数掉回「只剩己方持有的那些」再放行 barrier。同理,断言要用 `started` 数组**长度**,
`[...new Set(started)]` 会把双跑抹得干干净净。

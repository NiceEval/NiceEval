[NiceEval](https://niceeval.com/) CLI 是发现、运行和查看 eval 的入口。实际执行 eval 时采用 experiment-first 模型：**`exp` 选择可签入的运行配置**，experiment 后的位置参数才按 eval ID 前缀过滤。agent、model 和 flags 写在 `experiments/`，不靠临时 CLI 参数。

## 输出语言

[NiceEval](https://niceeval.com/) 的 CLI 和运行时文案支持本地化，不需要在 `niceeval.config.ts` 里加配置。需要固定输出语言时用环境变量：

```bash
NICEEVAL_LANG=en npx niceeval list
NICEEVAL_LANG=zh-CN npx niceeval list
```

检测顺序是 `NICEEVAL_LANG`、`NICEEVAL_LOCALE`、`LC_ALL`、`LC_MESSAGES`、`LANG`。以 `zh` 开头的值使用 `zh-CN`,其它语言使用 `en`;都没有时默认 `zh-CN`。这只影响终端/runtime 文案,不改变结果 JSON 里的机器字段,也不翻译 LLM judge prompt。

## `npx niceeval exp [group|config] [id-prefix...]`

```bash
# 运行 experiments/ 下全部实验
npx niceeval exp

# 运行一个实验组
npx niceeval exp compare-models

# 在该组里只运行 ID 以 weather 开头的 eval
npx niceeval exp compare-models weather
```

<Note>
  eval 过滤参数只出现在 experiment 选择之后。裸 `npx niceeval weather` 不会运行；请使用 `npx niceeval exp local weather` 或 `npx niceeval exp compare weather`。
</Note>

## `list`

```bash
npx niceeval list
```

用于检查 eval 发现、ID 和配置加载是否正常。

## `exp`

```bash
npx niceeval exp compare-models
npx niceeval exp compare-models weather-tool
```

运行命名 experiment，用矩阵比较 agents、models 或 flags。第二个参数开始是 eval ID 前缀过滤。

## `view`

```bash
npx niceeval view
```

打开本地结果查看器。默认查看最近一次 `.niceeval/` 运行。

## `--early-exit` 与 `--strict`

`--early-exit` 默认就是 `true`：某个 eval 的一次 attempt 通过后，会自动停止该 eval 剩余的 attempts（省钱）。所以显式传 `--early-exit` 没有效果，真正有用的是反过来关掉它——`--no-early-exit`，让 `--runs` > 1 时把每次 attempt 都跑完（测真实通过率而非提前收尾）。

`--strict` 不是"更严格地报错"，而是改变软阈值断言的判定：`.atLeast(n)` 这类软阈值（`soft` severity）平时失败不会把整条 eval 判为 `failed`（只是记一条不达标的断言），加了 `--strict` 之后，软阈值没达标也会让整条 eval 的 verdict 计为 `failed`。CI 中推荐加上，避免"断言分数不够但 eval 显示通过"的情况被放过。

## 环境变量覆盖

以下环境变量对应部分 flag，用于不便传 CLI 参数的场景（如 CI 环境配置）：

| 环境变量 | 对应 flag |
|---|---|
| `NICEEVAL_RUNS` | `--runs` |
| `NICEEVAL_MAX_CONCURRENCY` | `--max-concurrency` |
| `NICEEVAL_TIMEOUT` | `--timeout` |
| `NICEEVAL_BUDGET` | `--budget` |

优先级从高到低：CLI flag > 环境变量 > experiment / `niceeval.config.ts` 里的值 > 内置默认值。四个环境变量都要求合法数字，解析失败会直接报错退出。

[NiceEval](https://niceeval.com/) 还会在启动时自动加载 cwd 下的 `.env` 文件（不需要额外配置，也不覆盖已经存在的环境变量），常用于本地放 `ANTHROPIC_API_KEY` 这类鉴权信息，不用每次手动 `export`。

## 退出码

| 退出码 | 含义 |
|---|---|
| `0` | 全部 eval 通过（或 `list`、`init`、`clean` 等命令正常完成）。 |
| `1` | 存在 `failed` / `errored` 的 eval；或命令用法错误（如 `exp` 没匹配到任何实验、传了不支持的 flag 组合）。 |
| `2` | 运行时崩溃（未捕获异常），CLI 顶层兜底捕获后退出。 |
| `130` | 收到中断信号（Ctrl-C / SIGTERM）后退出。 |

需要 CI 把软阈值失败也当失败处理时，同时使用 `--strict`（见上）。

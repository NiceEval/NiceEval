接入你自己的应用只需要三个文件：一个 **adapter**（怎么调你的应用）、一个 **experiment**（评谁、跑几次）、一条 **eval**（断言什么）。下面是完整的最短路径；写给 coding agent 的一键接入和场景示例在后面。

## 最短路径：三个文件，一条命令

**0. 安装并初始化**（生成 `niceeval.config.ts` 和 `evals/` 目录）：

```bash
pnpm add -D niceeval
npx niceeval init
```

**1. adapter——怎么调你的应用。**对着你的应用本来就在对外的接口发一次消息，把回答放进一条 `message` 事件。最小接入只需要填 `status` 和 `events` 两个字段：

```ts
// agents/my-bot.ts
import { defineAgent } from "niceeval/adapter";

export default defineAgent({
  name: "my-bot",
  async send(input, ctx) {
    const r = await fetch(`${process.env.MY_BOT_URL}/chat`, {
      method: "POST",
      headers: { "content-type": "application/json" },
      body: JSON.stringify({ message: input.text }),
      signal: ctx.signal,
    });
    const body = await r.json();
    return {
      status: r.ok ? "completed" : "failed",
      events: [{ type: "message", role: "assistant", text: body.reply }],
    };
  },
});
```

**2. experiment——评谁：**

```ts
// experiments/my-bot.ts
import { defineExperiment } from "niceeval";
import myBot from "../agents/my-bot.ts";

export default defineExperiment({
  description: "my-bot 基线",
  agent: myBot,
  runs: 1,
});
```

**3. eval——断言什么，然后跑：**

```ts
// evals/refund-policy.eval.ts
import { defineEval } from "niceeval";
import { includes } from "niceeval/expect";

export default defineEval({
  description: "退款政策问答",
  async test(t) {
    await t.send("你们的退款政策是什么?");
    t.succeeded();
    t.check(t.reply, includes("30 天"));
  },
});
```

```bash
npx niceeval exp my-bot   # 跑起来
npx niceeval view         # 本地查看器里看结果
```

到这里第一条 eval 已经跑通。

<Warning>
这个最小 adapter 是**单轮**的：`t.send()` 可以连调多次，但每一轮都是独立的——第二轮不会记得第一轮说了什么，因为 adapter 还没告诉应用"历史怎么接上"。这不是 bug，是最短路径故意留白的第一个增量。你的应用接口收会话 id 的，透传 id 即可；服务端无状态的，由 adapter 拼接完整历史发过去——两种写法见[接入你的 Agent](/zh/guides/connect-your-agent)第三步。
</Warning>

接下来的每一步都是给 adapter 加可选增量：工具调用事件（解锁 `t.calledTool()`）、多轮会话（解锁跨轮记忆与 `t.newSession()` 隔离断言）、HITL、tracing——顺着[接入你的 Agent](/zh/guides/connect-your-agent) 逐层往下走即可，已写的 eval 不用改。

## 放进 CI

```yaml
name: evals
on: [pull_request]
jobs:
  evals:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npx niceeval exp ci --strict
```

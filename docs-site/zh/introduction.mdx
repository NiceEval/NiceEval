NiceEval 是一个受[eve](https://eve.dev)启发的 agent-native eval 工具。它追求低心智负担的 DX：大多数项目都可以在 10 分钟左右接入，写出第一条 eval，并开始稳定地检查 agent 行为。

它既能评给 Claude Code / Codex 写的 coding-agent 插件、Hook 和 Skill，也能直接评你自己的 AI agent 应用。无论你的 agent 基于 AI SDK、LangGraph、Pi 还是自研 loop，都可以通过 adapter 接进同一套 eval surface。

## 解决什么问题

像 Langfuse、Braintrust 这类工具更偏 tracing 和监控，对"写 eval、跑 eval、看结果"这条路径来说太重；NiceEval 从一开始就针对这条路径做了友好的开发体验。

## 整体架构

```text
   evals/*.eval.ts
        │
        ▼
   ┌─────────────────────┐
   │     niceeval        │
   └─────────────────────┘
        │
        │ Agent 适配器(官方)
        ▼
   ┌──────────────────────────────┐
   │        Docker Sandbox         │
   │   ┌────────────────────────┐  │
   │   │ Codex / Claude Code /  │  │
   │   │ 需要隔离工作区的应用    │  │
   │   └────────────────────────┘  │
   └──────────────────────────────┘
```

```text
   evals/*.eval.ts
        │
        ▼
   ┌─────────────────────┐
   │     niceeval        │
   └─────────────────────┘
        │
        │ Agent 适配器(官方，或者自己实现)
        ▼
   ┌──────────────────────────────┐
   │      你自己的 AI Agent        │
   │  (AI SDK · LangGraph · Pi ·   │
   │     自研 agent loop 等)       │
   └──────────────────────────────┘
```


## 两种接入模式

**Sandbox mode** 适合 Codex、Claude Code 这类必须在真实文件系统中改代码、跑命令的 coding agent。

**Direct mode** 适合你自己的 AI Agent，不需要 Docker。

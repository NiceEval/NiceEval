# 开发者文档与 DX 来源提取索引

检索与整理日期：2026-07-04

这组文件是对外部资料的来源级提取。每个文件尽量只总结对应来源自己的主张、结构和可迁移规则，避免把多个来源揉成一篇泛化总结。

## 已读取来源

- [01-diataxis-divio.md](./01-diataxis-divio.md)：Diátaxis 与 Divio Documentation System。重点是教程、How-to、参考、解释四类文档的信息架构。
- [02-google-style-guide.md](./02-google-style-guide.md)：Google developer documentation style guide。重点是风格层面的清晰、一致、可访问、全球化、步骤和代码示例。
- [03-github-docs.md](./03-github-docs.md)：GitHub Docs 的 best practices、content design principles、search、style guide、code annotations。重点是用户任务、可扫描性、内容模型和可发现性。
- [04-microsoft-style-guide.md](./04-microsoft-style-guide.md)：Microsoft Writing Style Guide。重点是简洁语气、开发者内容、reference 与 code examples、步骤写作。
- [05-write-the-docs.md](./05-write-the-docs.md)：Write the Docs guide。重点是文档为什么重要、README 基础、文档原则、docs-as-code 方向。
- [06-good-docs-project.md](./06-good-docs-project.md)：The Good Docs Project templates。重点是模板化内容类型和可直接套用的页面骨架。
- [07-api-docs-mdn-irbw.md](./07-api-docs-mdn-irbw.md)：MDN API reference 写作指南与 I’d Rather Be Writing API 文档课程。重点是 API 文档的 reference、overview、concept、guide、example 组合。
- [08-dx-sources.md](./08-dx-sources.md)：GetDX、Twilio、Phodal 的 DX 资料。重点是文档作为开发者体验的一部分，如何看待错误、文档、可发现性、交互式体验和度量。
- [09-niceeval-synthesis.md](./09-niceeval-synthesis.md)：把来源提取结果映射到 NiceEval 文档改造时的写作规则。

## 读取范围说明

这次不是只看搜索结果。已打开并读取了每个来源的主页面，以及与“开发者文档写作 / DX / 教程 / API 文档”直接相关的子页面或关键段落。

大型站点没有逐页通读全站，例如 Google style guide、GitHub Docs style guide、Write the Docs、I’d Rather Be Writing 都是资料库级站点；这里读取的是与当前任务相关的入口页、原则页、步骤页、示例页、内容模型页或 API 文档页。没有读取的全站内容不应被视为已总结。
